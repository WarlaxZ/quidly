  ```markdown
  | Light | Dark |

  | <img src="docs/screenshots/dashboard-light.png" width="420"> | <img src="docs/screenshots/dashboard-dark.png" width="420"> |
  ```
  then `sa105.png` and `planner.png` below (single images, `width="640"` or similar).
  3. **What it is** — free, self-hosted, UK-first; **money is integer pence end-to-end** (no float
  rounding); single-user per install; **not affiliated with HMRC — estimates, not tax advice.**
  4. **Features** — one line each: Bookkeeping (transactions, recurring rules, one-click bank-CSV
  import, multi-property); Receipt scanning (optional AI extraction with your own Anthropic key);
  SA105 & personal tax (£1,000 property allowance, Section 24 finance reducer, Scottish bands, SA105
  PDF export); Limited companies (corporation tax incl. marginal relief, dividends, director's loan
  s455 + benefit-in-kind); Plan ahead (what-if planner + salary-vs-dividend optimiser); Light & dark.
  5. **Self-host with Docker (recommended):**
  ```markdown
  ```bash
  git clone <your-fork> && cd quidly
  cp .env.example .env                                    # then set SESSION_SECRET
  docker compose run --rm quidly npm run set-password     # paste the Docker-Compose hash into .env
  docker compose up -d                                    # open http://localhost:3000
  ```
  ```
  Prose notes after the block:
    - **Password hash & the `$` gotcha:** docker-compose interpolates `$` in env values, so put the
      argon2 hash in `.env` with **each `$` doubled** — `set-password` prints exactly that
      "Docker-Compose" form (it prints three forms: dev backslash-escaped, Docker-Compose doubled,
      and raw).
    - **Data persists** in the `quidly-data` Docker volume (SQLite DB at `/data/quidly.db` + uploads
      at `/data/uploads`) across restarts and upgrades.
    - **`COOKIE_SECURE`:** leave unset (secure) when behind HTTPS; set `COOKIE_SECURE=false` **only**
      for plain-HTTP on a trusted LAN.
  6. **HTTPS reverse proxy (Caddy):**
  ```markdown
  ```caddyfile
  quidly.example.com {
      reverse_proxy localhost:3000
  }
  ```
  ```
  Note: Caddy auto-provisions a TLS certificate; served over HTTPS, leave `COOKIE_SECURE` unset.
  Traefik or nginx work equally well — point your proxy at `localhost:3000`.
  7. **Local development:**
  ```markdown
  ```bash
  npm install
  npm run set-password            # writes credentials — paste the DEV (backslash-escaped) hash into .env
  # .env also needs: SESSION_SECRET=... (openssl rand -base64 32) and DATABASE_URL="file:./dev.db"
  npx prisma migrate deploy       # apply migrations (Prisma v7: hand-authored SQL, not migrate dev)
  npx prisma db seed              # seed HMRC categories
  npm run dev                     # http://localhost:3000
  npm test                        # 198 tests
  ```
  ```
  One line noting Prisma v7 specifics: the datasource URL lives in `prisma.config.ts`, and migrations
  are applied with `migrate deploy` (not `migrate dev`).
  8. **Configuration** — a Markdown table with columns *Variable · Required · Description* covering
  `AUTH_USERNAME`, `AUTH_PASSWORD_HASH`, `SESSION_SECRET` (required); `DATABASE_URL`, `UPLOAD_DIR`
  (set by Docker; dev defaults `file:./dev.db` / `./uploads`); `COOKIE_SECURE` (optional, plain-HTTP
  LAN); `ANTHROPIC_API_KEY`, `EXTRACTION_MODEL` (optional, enables receipt scanning). Point at
  `.env.example` for the annotated source of truth.
  9. **Tech stack** — Next.js 16 (App Router) · Prisma v7 + SQLite (better-sqlite3 adapter) ·
  Tailwind v4 · iron-session + argon2id · Vitest · Docker.
  10. **Tax accuracy & caveats** — produces estimates, not advice; verify HMRC rates each April;
  currently configured for the **2025-26** tax year (other years fall back to 2025-26 with an in-app
  notice); single-tenant per install.
  11. **Licence** — AGPL-3.0 (see [`LICENSE`](LICENSE)). One short paragraph: open, but running a
  modified version as a network service means sharing your changes; commercial/hosted licensing from
  the author is possible.

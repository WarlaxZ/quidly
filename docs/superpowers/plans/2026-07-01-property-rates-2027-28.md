# Property-Income Rates + Tax Year 2027-28 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Introduce property-specific income-tax rates (E/W/NI 22/42/47% from 2027-28) via a regression-safe surcharge mechanism, configure the 2027-28 tax year (Scotland provisional), and surface it in the UI.

**Architecture:** Add `propertyRateSurchargeBps` + `provisional` to `TaxBands`. `estimatePropertyTax` keeps its tested marginal computation and *adds* `surcharge × taxable-property`. `financeCostReducer` takes the rate as a param (`2000 + surcharge`). Surcharge is 0 everywhere except E/W/NI 2027-28, so all existing results are byte-identical.

**Tech Stack:** TypeScript, Vitest. Money = integer pence; rates = integer basis points.

- [ ] **Step 2: Add the surcharge to `estimatePropertyTax` in `incomeTax.ts`.**
`estimatePropertyTax` currently is:
```ts
export function estimatePropertyTax(input: PropertyTaxInput): PropertyTaxResult {
  const { otherIncomePence, taxableProfitPence, financeReducerPence, taxYear, region } = input;
  const taxWith = incomeTaxOn(otherIncomePence + taxableProfitPence, taxYear, region);
  const taxWithout = incomeTaxOn(otherIncomePence, taxYear, region);
  const gross = taxWith - taxWithout;
  const taxOnPropertyPence = Math.max(0, gross - financeReducerPence);
  const marginalRate = taxableProfitPence > 0 ? gross / taxableProfitPence : 0;
  return { taxOnPropertyPence, marginalRate };
}
```
Replace with (keeps the marginal `base` verbatim, adds the surcharge):
```ts
export function estimatePropertyTax(input: PropertyTaxInput): PropertyTaxResult {
  const { otherIncomePence, taxableProfitPence, financeReducerPence, taxYear, region } = input;
  const bands = getBands(taxYear, region);
  // Base: ordinary-rate marginal tax on the property slice (unchanged behaviour).
  const base = incomeTaxOn(otherIncomePence + taxableProfitPence, taxYear, region)
    - incomeTaxOn(otherIncomePence, taxYear, region);
  // Property surcharge: uniform +Xbps across every property band (incl. top rate), so it is a flat
  // surcharge on the taxable portion of the property income (the part above the personal allowance).
  const surchargeBps = bands.propertyRateSurchargeBps ?? 0;
  const pa = effectivePersonalAllowance(otherIncomePence + taxableProfitPence, bands);
  const totalTaxable = Math.max(0, otherIncomePence + taxableProfitPence - pa);
  const otherTaxable = Math.max(0, otherIncomePence - pa);
  const taxableProperty = Math.max(0, totalTaxable - otherTaxable);
  const surchargeTax = Math.round(taxableProperty * surchargeBps / 10000);
  const gross = base + surchargeTax;
  const taxOnPropertyPence = Math.max(0, gross - financeReducerPence);
  const marginalRate = taxableProfitPence > 0 ? gross / taxableProfitPence : 0;
  return { taxOnPropertyPence, marginalRate };
}
```
(`effectivePersonalAllowance` and `getBands` are already in this module. For `surchargeBps === 0`, `surchargeTax` is 0 and the function is identical to before.)

- [ ] **Step 3: Parameterise `financeCostReducer` in `profit.ts`.**
Change the signature + body:
```ts
export function financeCostReducer(financeCostsPence: number, profitPence: number, reducerRateBps = 2000): number {
  const base = Math.max(0, Math.min(financeCostsPence, profitPence));
  return Math.round(base * reducerRateBps / 10000); // Section-24 reducer at the property basic rate
}
```
(Default `2000` preserves every existing caller/test.)

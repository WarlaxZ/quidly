# Tax Year 2026-27 + 2025-26 Corrections Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Configure the 2026-27 UK tax year across every engine and correct two confirmed 2025-26 rate errors, so Quidly computes 2026-27 figures natively (no fallback banner) and 2025-26 Scottish/BIK figures are right.

**Architecture:** Data-only. Each engine in `src/lib/tax/` holds a `Record<"YYYY-YY", …>` rate table; add a 2026-27 entry to each and one string to `CONFIGURED_TAX_YEARS`. Two 2025-26 entries are corrected. Tests are updated (corrections) and extended (new year). No engine control-flow changes.

**Tech Stack:** TypeScript, Vitest. Money = integer pence (`12_570_00`); rates = integer basis points (`2000` = 20%).

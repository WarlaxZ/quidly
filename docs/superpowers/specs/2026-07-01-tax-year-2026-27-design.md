# Tax Year 2026-27 + 2025-26 Rate Corrections — Design Spec

**Date:** 2026-07-01
**Status:** Approved (design); ready for implementation planning
**Phase:** 5 (rate maintenance)

## Problem & context

Quidly's tax engines each hold rate/threshold config in a `Record<"YYYY-YY", …>` keyed by tax-year
string, and `CONFIGURED_TAX_YEARS` (in `taxYear.ts`) lists the years the UI offers. Only `"2025-26"`
is configured; every other year (including the real current year 2026-27) falls back to 2025-26 rates
with an in-app "not configured" notice. The 2026-27 figures are now published, so we add that year.

While mapping the engines, **two pre-existing errors in the 2025-26 config were found and confirmed
against gov.uk**, and will be fixed in this cycle (user-approved scope):
1. **Scottish 2025-26 bands** use the 2024-25 starter/basic/intermediate boundaries.
2. **Director's-loan official rate of interest** is set to 3.75% for 2025-26; the correct 2025-26
   value is 2.25% (3.75% is the 2026-27 rate).

All figures below are verified against gov.uk / gov.scot / HMRC (see the research log in the plan).

**Constraint:** additive config + targeted corrections in `src/lib/tax/`. No structural/engine-logic
changes — only data (rate tables) and the tests that assert on them. Money stays integer pence, rates
integer basis points, per the existing idioms.

## Section 1 — Scope

**In:**
- Add `"2026-27"` to `CONFIGURED_TAX_YEARS` (`taxYear.ts`).
- Add a 2026-27 entry to each year-keyed engine: `bands.ts` (EWNI + Scotland), `dividendTax.ts`,
  `corporationTax.ts`, `nic.ts`, `directorLoan.ts`.
- Correct the 2025-26 Scottish bands (`bands.ts`) and the 2025-26 official rate (`directorLoan.ts`).
- Update/extend the affected tests to lock the new/ corrected values (`taxYear.test.ts`,
  `bands.test.ts`, `dividendTax.test.ts`, `corporationTax.test.ts`, `nic.test.ts`,
  `directorLoan.test.ts`, and any income-tax/scenario test whose 2025-26 Scottish figure shifts).

**Out:** the dedicated property-income rates (22/42/47%) that begin **2027-28** (not 2026-27 — noted
for a future cycle); modelling the ORI's new quarterly review (we use the 6 April start-of-year rate
as a single annual figure, consistent with the engine's design); any UI change (the year lights up
automatically via the existing config-year machinery); the £1,000 property allowance (unchanged,
not year-keyed — no change).

## Risks & caveats

- **Baseline-shifting corrections:** fixing the 2025-26 Scottish bands + official rate deliberately
  changes existing test expectations. Each changed expectation must be recomputed from first
  principles (not just "make the test pass") and the rationale noted, so we don't paper over a real
  regression. EWNI/CT/NIC 2025-26 figures must NOT change (guard against accidental edits).
- **Rate provenance:** every 2026-27 figure and both corrections are cited to gov.uk/gov.scot in the
  plan's research log; re-verify each April as rates move.
- **ORI simplification:** the official rate is reviewed quarterly from 2025; we model the 6-April
  start-of-year rate as the annual figure (documented; matches the single-rate BIK engine).
- **Data-only guarantee:** no engine control-flow changes; the config tables + their tests are the
  entire surface. The passing suite is the regression evidence.
